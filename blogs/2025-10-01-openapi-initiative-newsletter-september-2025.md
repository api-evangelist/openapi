<p>Welcome to the OpenAPI Initiative (OAI) September 2025 newsletter! We’ve had a break over the vacation season in the northern hemisphere, but are back to bring you initiative news, information on events and educational resources, and this time round, news of our v3.2 OpenAPI release!</p>
<h3 id="initiative-news">Initiative News</h3>
<p>It goes without saying that the big news &#8211; wait, <strong><em>enormous</em></strong> &#8211; news for this edition of the newsletter is two new releases of the OpenAPI Specification! We have a new minor release at <a href="https://spec.openapis.org/oas/v3.2.0.html">version 3.2.0</a>, which has helped us also deliver a patch version of 3.1 at <a href="https://github.com/OAI/OpenAPI-Specification/releases/tag/3.1.2">3.1.2</a>.</p>
<h4 id="versions-3.2.0-features">Versions 3.2.0 Features</h4>
<p>Version 3.2.0 brings together a feast of new features including:</p>
<ul>
<li>A brand new Tag Object structure, providing greater flexibility and richness in tagging objects.</li>
<li>Support for the <span style="font-family: monospace; font-weight: bold;">QUERY</span> HTTP method for implementing operations searching collections, plus the new <span style="font-family: monospace; font-weight: bold;">additionalOperations</span> keyword that allows HTTP methods not included in the Specification to be described.</li>
<li>Support for streaming data, which is a critical enhancement to support creating well-described APIs across so many use cases, including chat, AI, IoT, and financial services.</li>
<li>A new <span style="font-family: monospace; font-weight: bold;">querystring</span> keyword that allows all query parameters supported by an API to be described through a Schema Object.</li>
<li>Security Scheme enhancements, including support for OAuth 2.0 Device Authorization Flow and OAuth 2.0 Server Metadata.</li>
</ul>
<p>You can read more about the enhancement in our <a href="https://www.openapis.org/blog/2025/09/23/announcing-openapi-v3-2">blog post</a>, which includes both links to key resources such as the release itself and our comprehensive migration guide published on our Learn site.</p>
<p>Thanks go to the TSC and all contributors for this release, particularly <a href="https://github.com/handrews">Henry Andrews</a> and <a href="https://www.linkedin.com/in/lornajane/">Lorna Mitchell</a> for their significant contributions to getting this version over the line!</p>
<h4 id="moonwalk-update">Moonwalk Update</h4>
<p>Back at the start of the year we gave a <a href="https://www.openapis.org/blog/2025/02/05/moonwalk-2025-update">status update</a> on the progress of Moonwalk: its goals, desired outcomes, and what this Special Interest Group aims to achieve. In that post we said: “<em>The timeline for Moonwalk reaching a 4.0.0 release remains open-ended</em>”, and with the release of v3.2.0 what you are actually seeing is the first incremental step of the larger Moonwalk project. Many of the features of v3.2.0 were ideated through Moonwalk, and our focus is now on delivering more backwards-compatible incremental steps in the 3.x line. We will also keep an eye out for problems that show us that we <strong>need</strong> to break compatibility and make a 4.0 release, but we would like to discover that need through community feedback.</p>
<p>Key message is: <strong>Don’t wait for Moonwalk</strong>! Moonwalk is an ideas engine, and feeds the progress on the main development line of the OpenAPI Specification. It may come to pass that there is <strong>never</strong> a v4.0 of OpenAPI. If you were hanging on for v4.0, make the leap to v3.2 now.</p>
<h3 id="membership-news">Membership News</h3>
<p>We are pleased to welcome new members to the OpenAPI Initiative!</p>
<p><a href="https://jentic.com">Jentic</a> joined early 2025. Jentic is building the bridge between the AI World and the API World, providing agents with targeted, repeatable, and efficient workflows. Jentic agents are built on OpenAPI and Arazzo, making these specifications crucial building blocks in the Jentic platform. You can read more in our <a href="https://www.openapis.org/blog/2025/09/26/the-openapi-initiative-welcomes-jentic">interview</a> with <a href="https://www.linkedin.com/in/erikwilde/">Erik Wilde</a>, who as well as being OAI Ambassador is also Head of Enterprise Strategy at Jentic.</p>
<p>We were also joined in September by <a href="https://www.apideck.com/">Apideck</a>. Apideck is a Unified API provider, with an API that seeks to simplify integration across different SAAS platforms through one integration. <a href="https://www.linkedin.com/in/gertjandewilde/">Gertjan De Wilde</a> describes their motivation for joining as being <em>“..time to stand behind the spec that has enabled us to build our company.”</em>, which is of course fantastic news, and a great motivation for anyone looking for anyone thinking about becoming a member. We are looking forward to bringing you our new member profile on Apideck very soon!</p>
<p>If you are interested in membership we have held two breakfasts at Apidays (New York and London), where we introduce what membership means and take a look at the revised member benefits we are looking to offer. If you want to find out more, please check out our recent post on <a href="https://www.linkedin.com/feed/update/urn:li:activity:7378728523152388096">LinkedIn</a>, which includes our presentation from this event.</p>

<h3 id="ecosystem-spotlight">Ecosystem Spotlight</h3>
<p>The Ecosystem Spotlight for this edition comes from <a href="https://www.linkedin.com/in/shaneopenapi/">Shane O’Connor</a>, Go to Market Lead at OAI member <a href="https://scalar.com/">Scalar</a>.</p>
<p>Shane highlights the great work Scalar are doing improving parsing time with their OpenAPI parser:</p>
<p><em>Scalar has released a modern OpenAPI parser that’s gaining traction for its performance and comprehensive feature set. Written in TypeScript, <span class="citation">@scalar/openapi-parser</span> supports OpenAPI 3.1, 3.0 and Swagger 2.0, with support for the newly released OpenAPI 3.2 specification coming very soon. The parser is already trusted by teams at Mintlify &amp; Kong demonstrating its production readiness across diverse use cases, and offering highly significant performance improvements over existing parsers.</em></p>
<p><em>Beyond performance, <span class="citation">@scalar/openapi-parser</span> offers a comprehensive utility suite including reference dereferencing with tracking callbacks, document filtering, and automatic upgrading from Swagger 2.0 to OpenAPI 3.1 (soon to be 3.2). What distinguishes this parser is its plugin architecture for handling external references. This architecture enables a crucial differentiator: the parser works seamlessly on both server-side and browser environments, unlike many alternatives that are limited to Node.js. Developers can extend it with custom plugins to fetch definitions from databases, CDNs, or any data source. The onDereference callback provides visibility into schema resolution, invaluable for debugging complex multi-file specifications.</em></p>
<p><em>As OpenAPI documents grow in complexity and with OpenAPI 3.2 bringing new features, having a performant parser that handles everything from legacy Swagger 2.0 to the latest specification becomes critical for maintaining responsive development workflows. Scalar’s parser represents a modern solution that’s actively evolving alongside the OpenAPI specification itself.</em></p>
